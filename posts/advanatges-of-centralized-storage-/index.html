Advantages of Centralized Storage

By M. Awad

October 17, 2020

Disk Utilization

Thin provisioning

Thin provisioning provides a means by which makes the storage appear to the servers that they have more storage than you have paid.

Using centralized storage allows for thin provisioning; when the storage is getting low, adding more physical storage is a cinch. The client is oblivious to any change, which shifts your focus from the ‘just in case’ to ‘just in time’ model of purchasing storage space.

Deduplication and Compression

Detect and eliminates identical blocks; those blocks are then replaced with reference to a single copy of the block on disk, so it saves you a lot of space.

On the other hand, compression detects and eliminates redundant data and white space in files.

Resiliency

If any component failed there is another component to add as a backup.

Centralized management

It’s much easier to manage all your storage from a centralized location rather than separately across all your servers.

Storage Tiering

The system can have a wide range of storage media for different purposes, SSD (caters to hot data), SATA (caters to cold data) ..so on

Disaster recovery and load balancing

  • Replicate the data to a disaster recover site.
  • Allow for load balancing.

Virtualization support

One of the virtualization hallmarks is moving virtual servers between physical servers on the fly while they are still running. Using centralized storage unlocks the possibility of virtual machine migration.
